Created two CD workflows and merged them to main via PR #3 (all 3 CI checks green):

deploy-aws-lambda.yml:

- Copies app/, config.py, lambda_handler.py into the package directory
- Zips the package (excluding .pyc and __pycache__)
- Uses aws-actions/configure-aws-credentials@v4 with AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, AWS_REGION secrets
- Calls aws lambda update-function-code --function-name crescendo-api
- Polls aws lambda wait function-updated to confirm rollout

deploy-aws-amplify.yml:
- Builds the frontend with npm run build
- Waits 15s for Amplify's auto-triggered job (GitHub webhook) to appear
- Retrieves the latest job ID via aws amplify list-jobs (avoids LimitExceededException from calling start-job when a job is already running)
- Polls aws amplify get-job every 15s until status is SUCCEED, FAILED, or CANCELLED

Both workflows triggered successfully on the merge-to-main push; all 5 workflows (Backend Tests, Frontend Tests, Integration Tests, Deploy Backend to Lambda, Deploy Frontend to Amplify) completed with conclusion: success.
